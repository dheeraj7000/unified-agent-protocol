Create spec/v0.1/ Directory — 14 normative documents

P0
+ spec/v0.1/index.md + spec/v0.1/envelope.md + spec/v0.1/capability-card.md + spec/v0.1/task-graph.md + spec/v0.1/policy.md + spec/v0.1/context-contract.md + spec/v0.1/provenance.md + spec/v0.1/errors.md + spec/v0.1/events.md + spec/v0.1/approval-lifecycle.md + spec/v0.1/transport-http.md + spec/v0.1/transport-sse.md + spec/v0.1/security.md + spec/v0.1/compatibility.md
Problem: No normative specification exists. A third-party developer reading this repo cannot determine what a compliant UAP runtime must do. The existing docs/SPEC.md is explanatory prose, not a standard.
Required content per file
  • index.md: Protocol overview, goals, non-goals, RFC-2119 conformance language declaration, two conformance tiers: CORE (envelope + policy + events) and FULL (+ approval + SSE + version negotiation).
  • envelope.md: Normative rules — a runtime MUST reject any envelope missing uap, type, task_id, actor, or intent; MUST reject if actor.agent_id is absent; task_id MUST be unique within a server instance.
  • task-graph.md: A graph MUST NOT contain cycles. A runtime MUST reject graphs where depends_on references a node ID not in the graph. Independent nodes SHOULD execute concurrently.
  • policy.md: Enforcement order is fixed — (1) allowed_tools, (2) denied_tools, (3) risk level, (4) scope check, (5) approval gate. A runtime MUST apply all five in this order before invoking any capability.
  • events.md: task.accepted MUST be emitted before any tool.started. tool.completed MUST follow tool.started for the same node. The final event MUST be one of task.completed, task.failed, or task.cancelled. All events MUST include event_id, task_id, time, uap.
  • approval-lifecycle.md: On approval gate, runtime MUST emit approval.requested and set status to waiting_for_approval. Resume MUST be triggered by a separate client call. Runtime MUST emit approval.granted before continuing execution.
  • errors.md: Full taxonomy table — all 21 error codes with description, recoverable, and safe_retry values: INVALID_ENVELOPE, INVALID_ACTOR, INVALID_INTENT, INVALID_CONSTRAINTS, INVALID_CONTEXT_REQUEST, INVALID_POLICY, INVALID_EXECUTION, INVALID_EXECUTION_GRAPH, INVALID_RETURN, TOOL_NOT_ALLOWED, TOOL_DENIED, RISK_EXCEEDS_POLICY, MISSING_PERMISSION, APPROVAL_REQUIRED, TASK_CANCELLED, CAPABILITY_TIMEOUT, PLAN_GRAPH_CYCLE, PLAN_MISSING_DEPENDENCY, UNHANDLED_ERROR, UNSUPPORTED_VERSION, TASK_NOT_FOUND.
  • security.md: Threat model table — rows: prompt injection, capability escalation, confused deputy, data leakage (overbroad field return), replay attack (stale approval token), event spoofing, provenance tampering. Each row: threat description, UAP mitigation.
  • transport-http.md: All endpoints with method, path, request/response shape, HTTP status codes. POST /uap/tasks returns HTTP 200 even for status:failed — HTTP status reflects transport, not task outcome.
  • compatibility.md: Version format, unknown field handling (MUST preserve unknown top-level metadata fields), breaking-change definition, min_client_version semantics.

# conformance/runner.py — minimal skeleton
import httpx, json, sys
from pathlib import Path

def run(target: str, vectors_dir: str = "conformance/test-vectors") -> None:
    passed = failed = 0
    for f in sorted(Path(vectors_dir).glob("*.json")):
        v = json.loads(f.read_text())
        r = httpx.post(f"{target}/uap/tasks", json=v["input"]).json()
        ok = True
        if (exp := v["expect"].get("status")) and r.get("status") != exp:
            print(f"FAIL {v['name']}: expected status={exp}, got {r.get('status')}")
            ok = False
        if (ec := v["expect"].get("error.code")):
            got = r.get("error", {}).get("code")
            if got != ec:
                print(f"FAIL {v['name']}: expected error.code={ec}, got {got}")
                ok = False
        if ok:
            print(f"PASS {v['name']}"); passed += 1
        else:
            failed += 1
    print(f"\n{passed} passed, {failed} failed")
    sys.exit(1 if failed else 0)

if __name__ == "__main__":
    run(sys.argv[1] if len(sys.argv) > 1 else "http://localhost:8000")

# runtime.py — resume_after_approval
async def resume_after_approval(
    self, task_id: str, approver_id: str
) -> Dict[str, Any]:
    if task_id not in self._pending:
        raise UAPError(
            code="TASK_NOT_FOUND",
            message=f"No pending approval for task {task_id}",
            recoverable=False, safe_retry=False,
        )
    p = self._pending.pop(task_id)
    envelope, graph, completed = p["envelope"], p["graph"], p["completed"]
    cap_id = p["capability_id"]

    self.policy_engine.grant_approval(task_id, cap_id)
    await self.emit(envelope, "approval.granted", {
        "approver_id": approver_id, "capability_id": cap_id,
    })
    try:
        result = await self.execute_graph(envelope, graph, completed_so_far=completed)
        self.tasks[task_id] = {"status": "completed", "result": result}
        await self.emit(envelope, "task.completed", {"result": result})
        return {"task_id": task_id, "status": "completed", "result": result}
    except UAPError as exc:
        self.tasks[task_id] = {"status": "failed", "error": exc.to_dict()}
        await self.emit(envelope, "task.failed", exc.to_dict())
        return {"task_id": task_id, "status": "failed", "error": exc.to_dict()}
    finally:
        self.policy_engine.revoke_approvals(task_id)

def _detect_cycle(nodes: List[TaskNode]) -> bool:
    node_ids = {n.id for n in nodes}
    # 0 = unvisited, 1 = in-stack (gray), 2 = done (black)
    state: Dict[str, int] = {n.id: 0 for n in nodes}
    adj = {n.id: [d for d in n.depends_on if d in node_ids] for n in nodes}

    def visit(v: str) -> bool:
        state[v] = 1
        for u in adj[v]:
            if state[u] == 1:          # back edge = cycle
                return True
            if state[u] == 0 and visit(u):
                return True
        state[v] = 2
        return False

    return any(visit(n.id) for n in nodes if state[n.id] == 0)
